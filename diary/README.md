# Project Diary

This folder contains daily session logs tracking work done on the Unity Nodes project.

## Purpose

- **Daily records** of what was accomplished in each session
- **Decision history** capturing why we made certain choices
- **Progress tracking** to avoid repeating work or losing context
- **Learning log** documenting patterns, gotchas, and workflow improvements

## Format

Each entry follows the naming convention: `YYYY-MM-DD.md`

### Entry Template

```markdown
# YYYY-MM-DD — Brief Session Title

## Session Overview
- **Time**: Morning/Afternoon/Evening
- **Focus**: Main area of work
- **Status**: ✅ Complete / 🔄 In Progress / ⚠️ Blocked

## What We Did
- Bullet points of actual work completed
- Tools used, commands run
- Files created/modified

## Key Decisions
- Why we chose approach A over B
- Trade-offs considered
- Learnings captured

## Next Steps / Ideas
- Things to do next session
- Open questions
- Future improvements

## Notes
- Observations
- Gotchas encountered
- Patterns discovered
```

## Guidelines

- **Update after each session** before wrapping up
- **Be specific** about what was accomplished, not just what was discussed
- **Capture decisions** and the reasoning behind them
- **Note gotchas** immediately so future sessions avoid them
- **Cross-reference** related diary entries when building on previous work
- **Update the Index** table above when adding new entries

This diary complements CLAUDE.md (which documents *how* to work) by tracking *what* work was actually done.
