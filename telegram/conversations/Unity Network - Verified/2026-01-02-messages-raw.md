Something like this MacroDroid on Galaxy S9 runs as a foreground service with a persistent notification to stay alive, preventing system kills.  Use the Application Closed trigger to detect and relaunch your target app automatically.[reddit +1]
Prerequisites
Ensure MacroDroid has unrestricted battery optimization:
• Go to phone Settings > Apps > MacroDroid > Battery > Optimize battery usage > All apps > MacroDroid > Don’t optimize.
• Grant accessibility permission when prompted (required for reliable app detection).[macrodroidforum]
• Enable the persistent notification (it appears automatically; customize via MacroDroid home screen tile).[reddit]
Create the Macro
Follow these steps in MacroDroid:
1. Open MacroDroid and tap + Add Macro.[youtube +1]
2. Under Triggers, tap + > Applications > Application Launched/Closed. Select your target app and choose Application Closed. (Enable accessibility if prompted; avoid “Force legacy mechanism” unless issues occur.)[macrodroidforum +1]
3. Under Actions, tap + > Applications > Launch Application. Select the same target app.[youtube +1]
4. (Optional) Add a Wait Before Next Action (e.g., 1-2 seconds) under Actions to prevent rapid looping.[reddit]
5. Name the macro (e.g., “Keep App Alive”), tap the checkmark to save, and toggle it ON.[youtube]
Test and Troubleshoot
Close the target app manually; it should relaunch within seconds.  If unreliable, check S9’s Device Care > Battery > Background usage limits and exempt both apps, or add constraints like “Screen On” to the macro.  The persistent notification ensures MacroDroid survives reboots and doze mode.[cardata +1]
