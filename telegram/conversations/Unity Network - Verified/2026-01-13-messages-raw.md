🌐 Summary of the Communication Event
🧭 Purpose of the Session
Provide clarity and progress updates on Unity’s Scout and Runner task system.

Address misinterpretations from the previous communication event.

Walk through the user journey, expectations, and upcoming alpha testing phase.

🔍 Clarifications About Unity’s Readiness
Unity has been built and tested for over a year with telecom partners, carriers, and consultants.

“Building” refers not only to coding but also to:

Legal and regulatory frameworks

Telecom partner integrations

Task translation and delivery mechanisms

The platform is ready, the market is ready, and demand is present now.

Last week’s announcement was not the start of building — it was the start of opening opt‑in tasks to the community and gamifying the user experience to ensure compliance and confidence.

🧩 Overview of Scout & Runner Tasks
These tasks translate complex telecom verification processes into community‑powered actions.

🎓 Academy Mode
Both roles begin in Academy Mode to learn the what, how, and why behind each step.

Includes:

Knowledge modules

A quiz (like a driving theory test)

A practical simulation

Users must be approved by Unity’s admin team before performing real tasks.

Academy Mode remains available anytime for practice.

🛰 Scout Role (Discovery & Initial Verification)
Each country will have up to 50 scouts.

Scouts:

Discover and verify connectivity between their phone (origin) and Unity numbers (destination).

Must use a prepaid SIM or eSIM with international calling — this mimics telecom industry testing behaviour.

Submit the SIM/package details for admin approval before starting.

When calling Unity numbers:

Connectivity Verified → Unity IVR answers with a specific message.

No Connectivity → Any other outcome (disconnect, real person, etc.).

Scouts earn UPs for both outcomes, with higher rewards for verified connectivity.

Verified numbers are then sent to other scouts for secondary validation.

Once fully validated, the number is passed to runners.

🏃 Runner Role (Ongoing Verification)
Runners receive numbers that scouts have already verified.

They must:

Obtain the same SIM/eSIM package used by the scout.

Dial the number and confirm the Unity IVR still answers.

This creates a continuous validation loop.

If multiple runners fail to reach the Unity IVR, the system concludes the connectivity is lost.

Runners earn UPs based on:

Origin/destination

Per‑minute rates

Job issuer agreements

Scouts receive a small percentage of runner rewards for the connectivity they originally discovered.

🔄 Overall Task Loop
Discover → Verify → Execute → Re‑verify  
This cycle continues until connectivity disappears.

🧠 Why This Task Matters
It is a high‑value telecom service being translated into community‑powered proof‑of‑work.

With thousands of ULOs, Unity expects to build the world’s largest distributed connectivity verification network.

Understanding — not memorisation — is essential for compliance and safety.

🚀 Alpha Testing & Release Plan
A registration form for alpha testers will be released in the coming days.

Alpha testers will:

Be selected from multiple countries

Test the user interface and task translation

Provide data on clarity, usability, and performance

If testing goes smoothly, public release will accelerate.

The goal is not to delay — Unity wants Scout & Runner live as soon as possible.

🏁 Closing Notes
The speaker emphasises:

Confidence in the community’s ability to learn quickly

The importance of understanding over parroting

Commitment to frequent communication
